
### Next.js 기준으로 프롬프트 이용규칙을 작성해보려고 한다.
---

- 공식문서에 작성된 개발 권장 사항을 AI에게 먼저 요청하는 식으로 진행한다.

> Server-first 원칙과 최소 JS 번들을 우선합니다.
> 도메인 : 
> 요구: 
> 1) SEO가 중요하므로 SSR 기반 렌더링이 필요하다.
> 2) 데이터(fetch)와 UI(rendar)는 동일 Server Component 내에서 응집되도록 설정
> 3) React Query 등 클라이언트 상태관리 사용 금지
> 구현: 
> 	- app/books/page.tsx: getBookList()로 데이터 fetch + SSR 렌더
> 	- /ui/BookList.tsx: 리스트 UI
> 	- /api/getBookList.ts: 서버 전용 fetch 함수
> 품질기준
> 	- 빈/에러/로딩 상태 처리
> 	- next/image 최적화 
> 	- 파일병 분리 출력



### 데이터와 UI가 동일 Server Component 내에서 작성하는 이유
---


- 데이터의 흐름에 따라서 UI가 변하기 때문에 같은 컴포넌트에서 작성하는 것을 추천한다.
