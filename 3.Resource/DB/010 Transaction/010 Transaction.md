# 010 Transaction 
----

개요 : 서비스를 운영하면서 DB 성능 저하 이슈가 빈번히 발생했다. 그때마다 특정 쿼리만 골라 성능을 개선하는 방식으로 문제를 해결했지만, 시간이 지날수록 한계가 느껴졌다.
다수의 쿼리가 동시에 실행되거나, 중간에 실패했을 때 데이터가 꼬이는 현상을 겪으면서,
단순히 쿼리 튜닝이 아니라 트랜잭션 자체의 이해가 더 중요한 문제임을 깨달았다.
