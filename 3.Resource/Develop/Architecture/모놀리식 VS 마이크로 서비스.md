
### 모놀리식 vs 마이크로서비스 아키텍쳐 비교 


### 1. 모놀리식 구조 (Monolithic Architecture)

```
// 모든 기능이 하나의 애플리케이션에 포함
// 단일 배포 단위
// 하나의 데이터베이스
```

- 단일 JVM: 모든 컴포넌트가 같은 프로세스에서 실행
- 공유 메모리: EventBus가 메모리 내에서 이벤트 처리
- 직접 호출: 서비스 간 직접 메서드 호출
- 단일 배포: 전체 애플리케이션을 한 번에 배포

### 2. 마이크로서비스 구조 (Microservices Architecture)

```
// 각 기능을 독립적인 서비스로 분리
// 독립적인 배포 단위
// 각자 다른 데이터베이스
```

| 구분     | 모놀리식             | 마이크로서비스          |
| ------ | ---------------- | ---------------- |
| 구조     | 단일 애플리케이션        | 여러 독립 서비스        |
| 배포     | 전체 배포            | 서비스별 독립 배포       |
| 데이터베이스 | 단일 DB            | 서비스별 DB          |
| 통신     | 직접 호출            | HTTP/RPC/MQ      |
| 이벤트 처리 | 메모리 내 (EventBus) | 메시지 큐 (RabbitMQ) |
| 장애 격리  | 전체 영향            | 서비스별 격리          |
| 확장성    | 수직 확장            | 수평 확장            |
