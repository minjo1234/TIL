
1.쿠버네티스 기본 구조: 컨트롤 플레인 & 노드 

쿠버네티스는 크게 전체 클러스터를 관리하는 Control Plane(마스터)과 실제 컨테이너가 실행되는 Worker Node로 나뉩니다.

컨트롤 플레인 (Control Plane)
클러스터의 '두뇌' 역할을 하며, 전체적인 상태를 결정합니다.

- **kube-apiserver:** 모든 통신의 중심 창구입니다. CLI(kubectl)나 내부 컴포넌트 모두 API 서버를 통해 소통합니다.
- **etcd:** 클러스터의 모든 데이터를 저장하는 키-값(Key-Value) 저장소입니다. '진실의 원천'이라 불립니다.
- **kube-scheduler:** 새로 생성된 포드(Pod)를 어떤 노드에 배치할지 결정합니다.
- **kube-controller-manager:** 상태를 체크하고 '현재 상태'를 '바라는 상태'로 맞추는 역할을 합니다 (예: 복제본 유지).

워커 노드 (Worker Node)

실제 애플리케이션 서비스가 구동되는 곳입니다.

- **kubelet:** 각 노드에서 실행되는 에이전트로, 컨테이너가 포드 내에서 잘 작동하는지 관리합니다.
- **kube-proxy:** 노드의 네트워크 규칙을 관리하며 서비스 연결을 담당합니다.
- **Container Runtime:** 실제로 컨테이너를 실행하는 소프트웨어입니다 (Docker, containerd 등).
