
Next.js 14의 App Router에서 서버 컴포넌트가 도입되어 클라이언트 컴포넌트와 구분하여 사용이 가능한데, 서버 컴포넌트와 클라이언트 컴포넌트의 차이가 무엇일까?


서버 컴포넌트 - 기본적으로 Next.js가 사용하는 컴포넌트
(초기 로딩 속도 향상, 보안 측면 유리, 클라이언트(브라우저)로 전송되는 번들 크기를 줄인다.)

> "use server"  를 명시하지 않아도 사용할 수 있는 컴포넌트 이다.

서버 렌더링 장점을 디테일하게 (Benefit of Server Rendaring),

- Data Fetching (데이터 가져오기)
	- 서버에서 데이터를 가져가져 오기 때문에 성능이 향상된다.
	- 클라이언트 <-> 서버 (네트워크등과 같은 영향도를 받음)
	- Nextjs <-> Backend (대게 같은 데이터센터내라서 영향도가 적음)
- 보안 - 민감한 데이터와 로직을 서버에 보관
- 캐싱 - 서버에서 렌더링한 결과는 캐싱되어 여러 사용자에게 재사용가능, 이를 통해 조회 속도 향상 및 데이터의 재조회를 하지않아 비용 절감
- 성능 - 전체 웹이 클라이언트 컴포넌트로 구성되었을경우, 상호작용이 필요없는 UI 요소들을 서버 컴포넌트로 이동시키면 실행시켜야할 Javascript양이 줄어들기 때문에 
  (특히 인터넷 환경이나 성능이 낮은 장치를 사용하는 사용자에게 유리하다.)

- 스트리밍(Streaming) - 
- 검색 엔진 최적(SEO) - HTML을 반환해야함 검색엔진 최적화를 위해서는 (서버 컴포넌트는 서버에서 HTML을 반환해준다. 근데 정확히 언제사용하는지는 모름)
