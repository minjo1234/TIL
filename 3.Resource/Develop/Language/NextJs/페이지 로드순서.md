
- Next.js는 서버에서 React 컴포넌트를 실행해서 HTML을 만든다
- 이때 포함되는 데이터와 포함하지 않는 데이터만 기억해도 된다. (컴포넌트를 다 만들었는데 잘 작동하지 않는 부분이 있다면 해당 데이터가 mount 이후에 적용가능한 데이터인지 확인이 필요하다.)

- mount 시점에는 모든 초기 상태(useState 초기값)이 반연된다.
- mount 후에 추가되는 건 브라우저 전용 데이터 (localStorage, API 호출  결과)

렌더링


  1. Next.js는 React로 HTML을 만들고 화면에 보여줌
  2. Mount 이전 데이터 vs Mount 이후 데이터로 구분
  3. 브라우저 전용 데이터 = Mount 이후 (API, localStorage 등)
  4. 그 외 데이터 = Mount 이전에 렌더링
  5. 렌더링 = 화면에 HTML, 이미지 보여주는 과정

 Mount 과정:
  1. 컴포넌트 함수 실행 (렌더링)
  2. Virtual DOM 생성
  3. Virtual DOM → 실제 DOM 연결 ← 여기가 Mount!
  4. Mount 완료 표시
  5. useEffect 실행
