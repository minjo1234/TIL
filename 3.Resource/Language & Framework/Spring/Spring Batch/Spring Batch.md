- **정산 시스템:** 수백만 건의 결제 내역을 모아서 매일 밤 수익을 계산해야 할 때.
- **데이터 마이그레이션:** DB에 쌓인 수억 건의 데이터를 다른 포맷으로 변경해야 할 때.

Spring Batch 키 포인트 정리

  만약 이 프로젝트에 도입을 고려한다면, 핵심 개념은 다음과 같습니다:

  ★ Insight ─────────────────────────────────────

  1. 핵심 구조: Job → Step → (Chunk or Tasklet)

  Job (하나의 배치 작업)
   └─ Step 1 (Chunk 방식: Reader → Processor → Writer)
   └─ Step 2 (Tasklet 방식: 단일 작업 실행)
  - Chunk 방식: 대량 데이터를 일정 단위(chunk)로 읽고-가공-쓰기. 예: DB에서 1000건씩 읽어서 처리
  - Tasklet 방식: 단일 작업 실행. 예: 파일 삭제, API 호출 한 번

  2. 핵심 컴포넌트
  ┌───────────────┬─────────────────┬──────────────────────────────┐
  │   컴포넌트    │      역할       │             예시             │
  ├───────────────┼─────────────────┼──────────────────────────────┤
  │ ItemReader    │ 데이터 읽기     │ DB 조회, 파일 읽기           │
  ├───────────────┼─────────────────┼──────────────────────────────┤
  │ ItemProcessor │ 데이터 가공     │ 변환, 필터링, 유효성 검사    │
  ├───────────────┼─────────────────┼──────────────────────────────┤
  │ ItemWriter    │ 데이터 쓰기     │ DB 저장, 파일 생성           │
  ├───────────────┼─────────────────┼──────────────────────────────┤
  │ JobLauncher   │ Job 실행 트리거 │ 스케줄러, REST API에서 호출  │
  ├───────────────┼─────────────────┼──────────────────────────────┤
  │ JobRepository │ 실행 이력 관리  │ 메타 테이블에 성공/실패 기록 │
  └───────────────┴─────────────────┴──────────────────────────────┘
  3. 실무에서 중요한 포인트

  - 재시작(Restart): 실패 지점부터 재시작 가능 (ExecutionContext에 상태 저장)
  - 멱등성: 같은 Job을 다시 실행해도 안전하게 설계해야 함
  - chunk-size 튜닝: 너무 작으면 트랜잭션 오버헤드, 너무 크면 메모리 부담
  - 메타 테이블: BATCH_JOB_INSTANCE, BATCH_JOB_EXECUTION 등 6개 테이블이 필요 (자동 생성 가능)

  ─────────────────────────────────────────────────
