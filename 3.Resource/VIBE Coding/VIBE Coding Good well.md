구현 프롬프트 잘 쓰는 법

지금과 동일한 작업을 최소한의 변경사항으로 더 깔끔한 코드로 구현하려면,
어떻게 프롬프트하는게 더 좋았을지 자세한 키워드를 포함해 작성해주세요

- 관련 파일을 모두 상대경로로 참조해 나열하세요. 
- 최소한의 코드로 모듈화와 가독성이 높게 작성되어야합니다.
- 기존 코드베이스의 컨벤션과 구조를 지키며 구현되어야합니다.

디버깅 하는법 

버그를 정확히 수정하기위해선,
당연히 버그를 정확하게 설명해줘야합니다.

프롬프트에 아래 4가지 정보가 포함됐는지 점검해보세요.

- Where: 접속한 주소
- Try: 시도한 행동(아무 행동하지 않은 경우 생략가능)
- Expected
